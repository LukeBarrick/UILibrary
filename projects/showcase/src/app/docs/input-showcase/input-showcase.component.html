Form Field & Input Components

The uilibrary2-form-field component works together with the ui-input directive to provide consistent, accessible form inputs with support for labels, prefixes, suffixes, and error states.

Error States

Form fields can display error states for validation feedback.

Error state is toggled by: "{{textInputError}}" (empty string shows error)

Textarea Inputs

Form fields also work with textarea elements for multi-line text input.

API Reference

Component/Directive Purpose Usage
uilibrary2-form-field Wrapper component for form inputs <uilibrary2-form-field>...</uilibrary2-form-field>
ui-label Directive for field labels <label ui-label>Field Name</label>
ui-input Directive for input and textarea elements <input ui-input type="text">
ui-prefix Directive for prefix elements <div ui-prefix>Content</div>
ui-suffix Directive for suffix elements <div ui-suffix>Content</div>
ui-text-prefix Directive for text-specific prefix styling <div ui-text-prefix>$</div>
ui-text-suffix Directive for text-specific suffix styling <div ui-text-suffix>%</div>
