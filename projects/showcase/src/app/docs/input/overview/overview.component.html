Input

Overview

The [ui-input] directive, implemented by InputComponent, is a directive-based form control that plugs into the FormFieldComponent via Angular content projection. It provides native <input>-like behavior and integrates with both Angular Forms APIs and the UIFormFieldControl<T> interface to support a standardized UI and reactive validation feedback.

👷 Directive Selector

[ui-input]

Attach this directive to a native <input> element to automatically wire it into a <uilibrary2-form-field> wrapper. Example: 

<uilibrary2-form-field>
  <label ui-label>Username</label>
  <input type="text" ui-input [(ngModel)]="username" />
</uilibrary2-form-field>

🧠 Integrated Interfaces

  • UIFormFieldControl<string> — exposes the form control state to FormFieldComponent.
  • ControlValueAccessor — allows use with ngModel, formControl, and formControlName.

🧩 Responsibilities

  • Generates a unique ID via UUIDService for accessibility/labeling.
  • Handles focus, blur, and input events.
  • Maintains internal focus and empty state flags for UI label floating.

📦 Inputs & Outputs

Type Name Description
{{'@'}}Input disabled Sets the disabled state of the control.

🔁 Form Control State Mapping

Property Usage
shouldLabelFloat true when the input has focus or value is not empty.
hasErrors true if control is invalid.
touched true if control has been blurred by the user.
dirty true if the user has modified the value.
empty true if no value has been entered.

💡 Implementation Details

  • Applies the disabled state directly via Renderer2.
  • Reads/writes input value using ElementRef for native control interoperability.
  • Focus state is tracked internally via {{'@'}}HostListener decorators.
  • Form control state is pulled from NgControl when used within Angular Forms.

📌 Example: Complete Usage with Floating Label

<uilibrary2-form-field>
  <label ui-label>Email Address</label>
  <input
    ui-input
    type="email"
    [disabled]="isDisabled"
    [(ngModel)]="email"
/>
</uilibrary2-form-field>
