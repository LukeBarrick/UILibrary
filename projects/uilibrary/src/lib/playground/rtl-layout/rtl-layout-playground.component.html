Active Direction

The [uiRtlLayout] directive sets a dir attribute on the host element. All CSS logical properties and FlexBox inside that element respond automatically — no direction-specific overrides needed.

1 · FlexBox Row Reversal

A flex-direction: row container reverses item order in RTL without any extra CSS — the browser uses the dir attribute.

@for (item of flexItems; track item.label) {
{{ item.label }}
}

Switch to RTL — the boxes reverse without touching flex-direction.

2 · margin-inline-start

margin-inline-start maps to margin-left in LTR and margin-right in RTL.

@for (key of spacerKeys; track key) {
mis-{{ key }}
Content block
}

Watch the indent shift sides when you switch to RTL.

3 · padding-block-end

padding-block-end maps to padding-bottom in horizontal writing modes. It is not affected by LTR/RTL but demonstrates the block axis.

@for (key of [1, 2, 3, 4, 5]; track key) {
pbe-{{ key }}
}

4 · border-inline-start

An accent border on the start side of an element — left in LTR, right in RTL. Useful for callout cards, active menu items, and quotes.

5 · Logical Text Alignment

.text-align-start aligns to the start side — left in LTR, right in RTL. Contrast with physical text-align: left.

text-align: left (physical)

This text is always left-aligned, regardless of direction.

text-align-start (logical)

This text aligns to the start edge — left in LTR, right in RTL.

text-align-end (logical)

This text aligns to the end edge — right in LTR, left in RTL.

6 · Icon + Label with margin-inline-start

8 · Text Input & Form Field

✅ Compatible

The native <input> element inherits the dir attribute — text entry, cursor position, and placeholder alignment all adapt automatically. The floating label now uses inset-inline-start so it snaps to the correct edge in both directions.

9 · Select Dropdown

⚠️ Partial

Internal ng-select CSS has been migrated to logical properties (value position, input padding). The dropdown panel position is set by the ng-select library via JavaScript (getBoundingClientRect) and cannot be overridden with pure CSS — it may not align perfectly at the viewport edges in RTL.

10 · Status Tags

✅ Compatible

Status tags are plain wrappers with colour-only styling. Text content flows naturally with the dir attribute and the flex row reverses in RTL.

11 · Checkbox

✅ Compatible

Checkboxes now use logical utility classes (mis-4 / mie-4) for label spacing and inset-inline-start for the check indicator position. Both LTR and RTL render correctly.

12 · Toggle

✅ Compatible

The toggle slider knob now uses inset-inline-start: 0 for initial position. In RTL mode a scoped [dir="rtl"] rule applies translateX(-15px) so the knob animates toward the start edge. Label spacing uses the same logical utilities as the checkbox.

14 · Date Picker

⚠️ Partial

The date input fields inherit the dir attribute so text entry and cursor behaviour flip correctly. The calendar popup panel itself uses absolute positioning anchored to inset-inline-start 0 after the fix to date-picker/_index.scss. Month/day label order within the calendar header depends on the Angular CDK date locale and is not affected by dir alone.
