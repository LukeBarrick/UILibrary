RTL Layout

Active Direction

The [uiRtlLayout] directive sets a dir attribute on the host element. All CSS logical properties and FlexBox inside that element respond automatically — no direction-specific overrides needed.

1 · FlexBox Row Reversal

A flex-direction: row container reverses item order in RTL without any extra CSS — the browser uses the dir attribute.

2 · margin-inline-start

margin-inline-start maps to margin-left in LTR and margin-right in RTL.

Watch the indent shift sides when you switch to RTL.

3 · padding-block-end

padding-block-end maps to padding-bottom in horizontal writing modes. It is not affected by LTR/RTL but demonstrates the block axis.

@for (key of [1, 2, 3, 4, 5]; track key) {
pbe-{{ key }}
}

4 · border-inline-start

An accent border on the start side of an element — left in LTR, right in RTL. Useful for callout cards, active menu items, and quotes.

5 · Logical Text Alignment

.text-align-start aligns to the start side — left in LTR, right in RTL. Contrast with physical text-align: left.

text-align: left (physical)

This text is always left-aligned, regardless of direction.

text-align-start (logical)

This text aligns to the start edge — left in LTR, right in RTL.

text-align-end (logical)

This text aligns to the end edge — right in LTR, left in RTL.
