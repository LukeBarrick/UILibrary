Form Field

Overview

The FormFieldComponent is a flexible and reusable component that acts as a wrapper for native form control elements (i.e <input>, <textarea>, custom input elements), providing a consistent structure and styling behavior. This component supports content projection for prefixes, suffixes, and labels and integrates naturally with the control's validation and interaction state.

✅ Supported Controls

  • <input> – text, number, password, email, etc.
  • <textarea>
  • Any custom control implementing UIFormFieldControl<T>

🚫 Excluded Controls

  • <input type="checkbox">
  • <input type="radio">

These types have separate, dedicated implementations due to unique UX and accessibility requirements.

🧩 Content Projection Slots

Selector Purpose Example
[ui-prefix] Icon/element prefix <mat-icon ui-prefix>phone</mat-icon>
[ui-text-prefix] Text prefix <span ui-text-prefix>value</span>
[ui-label] Label element <label ui-label>Email</label>
[ui-input] Main control <input ui-input type="text" />
[ui-suffix] Icon/element suffix <button ui-suffix>...</button>
[ui-text-suffix] Text suffix <span ui-text-suffix>USD</span>

🧠 Internal Behaviors

  • .form-field-disabled — Applies when the control is disabled
  • .form-field-label-float — When label should float
  • .form-field-errors — When control has errors
  • .form-field-touched — When control has been touched
  • .form-field-dirty — When control has been modified

The component also exposes a focusControl() method for managing internal focus on click.

🧪 Required Interface

Custom inputs projected into this component must implement the following interface:

export interface UIFormFieldControl<T> {{ '{' }}
  stateChanges: Observable<void>;
  shouldLabelFloat: boolean;
  disabled: boolean;
  hasErrors: boolean;
  touched: boolean;
  dirty: boolean;
{{ '}' }}

🛠 Usage Examples

1. Basic Text Input
<uilibrary2-form-field>
  <label ui-label>Email</label>
  <input ui-input type="email" [(ngModel)]="email" />
</uilibrary2-form-field>
2. With Icon Prefix and Suffix Button
<uilibrary2-form-field>
  <mat-icon ui-prefix>email</mat-icon>
  <label ui-label>Email</label>
  <input ui-input type="email" />
  <button ui-suffix (click)="clear()">Clear</button>
</uilibrary2-form-field>
3. Text Prefix/Suffix
<uilibrary2-form-field>
  <span ui-text-prefix>$</span>
  <label ui-label>Amount</label>
  <input ui-input type="number" />
  <span ui-text-suffix>USD</span>
</uilibrary2-form-field>
4. Custom Component Input
<uilibrary2-form-field>
  <label ui-label>Search</label>
  <my-custom-search ui-input></my-custom-search>
</uilibrary2-form-field>

🧼 Style Recommendations

Ensure your global styles or SCSS include styling for these structural classes:

  • .form-field
  • .form-field-prefix
  • .form-field-text-prefix
  • .form-field-suffix
  • .form-field-text-suffix
  • .form-field-disabled, .form-field-label-float, etc.
